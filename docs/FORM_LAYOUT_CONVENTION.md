# 폼 테이블 배치 규칙

> 모든 등록·수정·상세 화면(FormTable 사용 폼)이 따라야 하는 통일된 행 순서.
> 샘플 구현: [AuditPlanTab.tsx](../frontend/src/components/ehs/AuditPlanTab.tsx)

## 흐름 (PC / Mobile 공통, 위에서 아래로)

| # | 행 구성 | 비고 |
|---|---------|------|
| 1 | 제목 \| 유형(셀렉박스1) | 좌우 2분할 |
| 2 | 셀렉박스2(예: 부서) \| 상태 | **상태는 상세에서만 표기**. 등록/수정에서는 좌측을 풀폭으로 |
| 3 | 도메인 고유 필드들 | 모델에 있는 식별자·도메인 식별 필드 (예: 계획번호, 감사원). 풀폭 또는 2분할 |
| 4 | 시작일 \| 종료일 | 날짜 페어가 있는 경우, 감사원 등 도메인 필드 바로 아래 |
| 5 | 목적 | 풀폭 |
| 6 | 비고 | 풀폭, **목적 바로 아래** |
| 7 | 작성자 \| 생성일 | |
| 8 | 수정자 \| 수정일 | **등록 모드에서는 비표시**. 상세는 `modifiedAt !== createdAt` 일 때만 |
| 9 | 계획승인자 \| 완료승인자 | 결재 패턴이 있는 폼만 |
| 10 | 체크리스트 | 풀폭, 마지막 행 |

## 규칙

### 1. 표시·생략 원칙
- **상태**: 상세에서만 표시. 등록/수정에는 행 자체를 두지 않음 (좌측 단일 필드는 풀폭 처리)
- **수정자 라인**: 등록 모드에서는 완전히 숨김. 수정 모드와 상세에서만 노출
  - 상세에서는 추가로 `modifiedAt`이 `createdAt`과 다를 때만 (수정 이력 존재 시에만)
- **작성자/수정자 행에 "부서" 컬럼은 두지 않음** — 이름과 시점(생성일/수정일)만 표시
- **모델에 없는 필드는 행 자체를 만들지 말 것** (`—`나 빈 셀로 자리 유지 금지)

### 2. 도메인 필드 배치
- 시작일/종료일은 항상 감사원·담당자 등 사람 필드 **아래**에 위치
- 목적 → 비고 순서는 **연속**되어야 함. 사이에 다른 필드 끼우지 말 것
- 도메인 고유 필드(측정값·점검결과 등)는 시작일/종료일과 목적 **사이** 또는 식별자(번호) 위치에 묶음 배치

### 3. 셀 너비/스타일 (PC)
- 라벨 셀: `labelSx` = `{ width: 140, minWidth: 140, fontWeight: 'bold', bgcolor: 'grey.100', justifyContent: 'center', textAlign: 'center' }`
- 값 셀: `valueSx` = `{ flex: 1, px: 2, py: 1.5 }`
- 좌측 값 셀(우측에 셀이 더 있을 때): `valueBorderSx = { ...valueSx, borderRight: 1, borderColor: 'grey.300' }`
- 풀폭 행: 라벨 1개 + 값 1개(`valueSx`)로 끝

### 4. 모바일
- 각 항목을 단일 라벨 + 단일 값 박스로 세로 적층
- 라벨 스타일: `{ bgcolor: 'grey.200', px: 1.5, py: 0.75, borderRadius: 0.5, fontWeight: 'bold' }`
- PC와 **동일한 행 순서**를 유지할 것

### 5. 필드를 새로 추가/제거할 때
폼에서 어떤 필드를 빼거나 새로 넣어야 한다면, **반드시** 다음을 동시에 처리:
1. 프론트 TypeScript 타입 (`*.types.ts`)
2. 백엔드 모델 (Java)
3. Mapper XML (resultMap / INSERT / UPDATE)
4. 서비스 (필드 전파, change-tracking)
5. Flyway 마이그레이션 (`Vxxx__설명.sql`)
6. 다른 페이지/탭에서 이 필드를 참조하는 곳

DB 컬럼만 살아있고 UI에서 가린 상태로 두지 말 것 — 잔존 컬럼은 잠재 버그·혼동의 원인.

## 예시 (감사 계획 — AuditPlanTab)

**PC 상세:**
```
제목 | 유형
부서 | 상태
계획번호 (풀폭)
감사원 (풀폭)
시작일 | 종료일
목적 (풀폭)
비고 (풀폭)
작성자 | 생성일
수정자 | 수정일   ← 이력 있을 때만
계획승인자 | 완료승인자
체크리스트 (풀폭)
```

**PC 등록/수정:**
```
제목 | 유형
부서 (풀폭, 상태 없음)
감사원 (풀폭)
시작일 | 종료일
목적 (풀폭)
비고 (풀폭)
작성자 | 생성일
수정자 | 수정일   ← 수정 모드일 때만
계획승인자 | 완료승인자
체크리스트 (풀폭)
```
